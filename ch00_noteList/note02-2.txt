* export, import
  - 목적: 코드 관리, 유지/보수
  - export는 기능을 제공, import는 다른 모듈을 사용

 * export
  - 목적: 다른 모듈에게 심벌의 의미를 전달
  - interface, class, type, let, const 등 앞에 사용
 
 * import
  - import { 심벌 목록 } from '파일 상대 경로'
  - impert * as 심벌 from '파일 상대 경로'

 * export default
  - 목적: 자바스크립트와 호환하기 위해
  - 한 모듈의 기능 중 '무조건 1개'만 사용
  - import 사용 시, 중괄호 {} 없이 사용
  - export 등이 있는 파일에서도 사용

  * chance, ramda 등 외부 패키지 설치
   - npm i -S chance ramda                  <실행할 때>
   - npm i -D @types/chance @types/ramda    <개발할 때만>
